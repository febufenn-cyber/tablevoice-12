## Factuality rules

The draft must never claim that:

- an investigation occurred unless confirmed;
- the customer was contacted unless confirmed;
- a refund or replacement was issued unless confirmed;
- the restaurant checked CCTV, order logs, or staff records unless confirmed;
- the allegation is false without evidence;
- a named employee was disciplined;
- the customer’s identity or order details are known when they are not.

## Issue-specific posture

| Issue | Default posture | Required action before reply |
|---|---|---|
| Positive praise | Thank, reference a real detail, invite return | None |
| Mixed 3-star review | Acknowledge both positive and negative elements | Check for factual assumptions |
| Slow service | Acknowledge wait; avoid excuses | Request visit/time context when useful |
| Food quality | Acknowledge experience; do not dispute taste | Check item and preparation context |
| Delivery failure | Distinguish restaurant and delivery partner carefully | Check order timeline if available |
| Pricing criticism | Explain value only if approved | Do not shame customer |
| Hygiene concern | Escalate | Owner/manager review required |
| Allergy/food safety | Stop normal drafting | Red workflow required |
| Staff accusation | Protect privacy and escalate | Manager review required |
| Suspected fake review | Stay factual; avoid public accusation | Verify records and platform-reporting option |

## Learning from edits

For every recurring edit, record:

- Original wording
- Owner wording
- Reason
- Restaurant-wide rule or one-off exception
- Effective date
- Approver

Do not silently learn a global rule from one edit.
