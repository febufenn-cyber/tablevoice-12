# Phase 1 Pilot Runbook

1. Confirm the restaurant passed the Phase 0 entry gate.
2. Create the organisation and restaurant.
3. Activate a voice profile with compensation and employee-name policies.
4. Import no more than ten historical reviews initially.
5. Verify every imported row against its source.
6. Process green, amber, and synthetic red drills before live sensitive cases.
7. Run QA before sending any draft for approval.
8. The restaurant manually posts approved replies.
9. Record publication evidence or the reason it was not posted.
10. Assign and close internal actions separately from the public reply.
11. Generate one weekly report and review it manually before delivery.
12. Compare operator minutes, acceptance, publication, and incidents against Phase 0.

Stop the pilot for an affected restaurant after a critical incident until containment, root cause, correction, and resume approval are documented.
